一、项目概述与建设目标

本系统面向城市更新规划决策，以建成区为分析范围，综合交通、环境、设施、政策、宗地形态、地形及人口等多源空间数据，通过栅格叠置 + 宗地关联的方式，生成全域更新潜力评价成果，并支持人工校核验证后输出专题图与报表。

系统核心逻辑分为三大阶段：

1

预处理阶段

坐标统一 · 范围裁剪 · 数据质检

2

更新评价阶段

因子量化 · 栅格叠置 · 宗地赋值

3

验证与输出阶段

人工校核 · 权重调整 · 成果出图

预期成果：

  • 成果图1：全域更新潜力评价图（栅格/面状分级设色）
  • 成果图2：宗地更新潜力分析图（潜力值关联至宗地面要素）
  • 可选：分阶段叠合图、片区指引标签图

五、核心分析算法详解

评价体系采用目标层 → 准则层 → 影响要素 → 具体指标四级结构。各指标通过空间分析生成栅格得分面，再逐级加权叠置。

5.1 更新动力性分析（正向驱动因子）

动力性反映「为什么要更新」，分值越高代表更新动力越强。准则层默认权重如下（可在 M8 配置）：

准则层默认权重影响要素指标得分规则
交通便捷度 30% 轨交通达性 单线地铁站点 300m→3分，600m→2分，1000m→1分（缓冲区）
轨交通达性 两线及以上地铁站点 300m→4分，600m→3分，1000m→2分
区位优越性 路网可达性 / 交通设施 / CBD 路网可达性 1-5 分；交通设施缓冲300m→1分；CBD缓冲1000m→3分
环境舒适度 20% 生态舒适性 重要生态廊道 建成区内河道/绿廊缓冲 500m → 2分
绿地丰富性 开敞空间 + 现状绿地 大型绿地/湖泊缓冲500m→2分；现状绿地缓冲300m→1分
设施完善度 25% 公服便利性 市级公服 + 便民设施 市级公服缓冲1000m→2分；便民设施缓冲300m→1分
商业发达性 市级商业设施 缓冲 1000m → 1分
政策支持度 25% 战略发展区 发展圈带 / 战略片区 / 近期重点区 圈带→1分；战略片区→1分；近期重点区→2分

5.1.1 轨交通达性 — 多级缓冲区分析

原理：以地铁站点为源，按 300/600/1000m 半径生成多级缓冲区，不同级别赋予不同得分，取最大值合并为栅格。

ArcEngine 实现：

  • 接口：ITopologicalOperator.Buffer() 生成矢量缓冲面
  • 或 GP 工具：Multiple Ring Buffer 一步生成多环缓冲
  • 面转栅格：FeatureToRaster，值字段 = 得分
  • 多环合并：RasterMapAlgebraOp 取 MAX 合成单一轨交得分栅格

5.1.2 路网可达性 — 网络分析

原理：基于路网数据集，计算各栅格单元到城市中心/副中心的通行成本，成本越低得分越高（1-5 分）。

ArcEngine 实现：

  • 构建 INetworkDataset，设置通行成本属性（道路等级/限速）
  • 使用 INASolverODCostMatrix 或 INAAgentSolverCostDistance
  • 输出成本距离栅格，通过 Reclassify 映射为 1-5 分
  • 备选简化方案：无 Network Analyst 许可时，用欧氏距离 + 道路密度栅格近似

5.1.3 环境/设施因子 — 缓冲区 + 欧氏距离

原理：对点/线/面要素按规则距离缓冲，缓冲范围内赋固定分值；多个因子栅格取 MAX 或 SUM（按指标设计）。

ArcEngine 实现：

  • 点要素：PointDistance 或 EuclideanDistance 生成距离栅格 → 重分类
  • 面要素：Buffer → FeatureToRaster
  • 线要素（生态廊道）：Buffer(500) → 面转栅格

5.1.4 政策支持度 — 空间包含检测

原理：判断评价单元是否落在政策红线范围内，采用 Identity 或 SpatialJoin 叠加检测。

ArcEngine 实现：

  • IGeoProcessor.Execute("Identity", ...) 叠加政策区与评价格网
  • 按重叠情况赋值：近期重点区 2 分 > 战略片区 1 分 > 发展圈带 1 分
  • 面转栅格输出政策得分面

5.1.5 动力性准则层合成

动力性得分 = Σ (准则层权重i × 准则层得分i)
准则层得分i = Σ (指标得分j)  （同准则层内各指标得分相加，上限按设计封顶）

通过 RasterMapAlgebraOp 或 WeightedOverlay（GP 工具 Weighted Overlay）实现栅格加权求和，输出 动力性栅格（值域建议标准化为 0-100）。

5.2 更新可行度分析（约束与基础条件因子）

可行度反映「能不能更新」，包含正向因子（人口密集）和负向因子（地形限制、破碎度高）。

影响要素具体指标得分算法要点
宗地情况 斑块破碎度 PD -2 PD 越高（地块越碎）扣分越多，通过 FS 指数映射
形状指数 SI -1 形状越不规则扣分，SI 越大得分越低
基地情况 高度（DEM） -1 高程超过阈值区域扣分
坡度（DEM） -1 坡度 > 15° 区域扣分
人口情况 人口密度栅格 +2 人口密度越高得分越高（更新需求大）

5.2.1 斑块破碎度 PD 与形状指数 SI

原理（来源：03 宗地分析.pdf）：

SI（形状指数）= P / (2 × √(π × A))   （P=周长，A=面积；圆形 SI=1，越不规则越大）
FS（综合破碎度）= 1 − 1/ASI   （ASI = Σ(SIi × Ai) / TA）
PD（斑块密度）= NP / TA   （NP=斑块数，TA=总面积 hm²）

ArcEngine 实现：

  • 对土地利用斑块执行 Dissolve（按 landcode 融合相邻同类型斑块）
  • AddGeometryAttributes 计算面积、周长
  • ICursor 遍历要素，CalculateField 计算 SI 字段
  • Statistics 按地类汇总 → 计算 PD/MPS/ED/FS
  • 将 FS 值关联回斑块面 → 面转栅格（PD 高区域 → -2 分）
  • SI 值直接映射：SI 1.0-1.3 不扣分，SI > 3.0 扣 -1 分

5.2.2 DEM 高程与坡度

ArcEngine 实现：

  • 已有 DEM 栅格 → GP Slope 生成坡度栅格
  • Reclassify：高程 > 设定阈值（如 50m）→ 值 = -1，否则 = 0
  • Reclassify：坡度 > 15° → 值 = -1，否则 = 0

5.2.3 人口密度

原理：人口密度越高，更新需求和商业价值越大，为正向因子。

ArcEngine 实现：

  • 加载人口栅格（参考 04 人口数据.png 热力分布）
  • Reclassify 按自然断点分为 3-5 级，最高级 → +2 分，其余 → 0-1 分
  • 裁剪至建成区范围

5.2.4 可行度合成

可行度得分 = 人口得分 + 地形得分（高程 + 坡度）+ 宗地得分（PD + SI）
注：负向因子直接扣分，最终可行度栅格值域可能为负，后续需标准化处理

5.3 综合潜力评价与等级划分

更新潜力总分 = 动力性得分 × W动力 + 可行度得分 × W可行
默认：W动力 = 0.7，W可行 = 0.3（可配置）
潜力等级分值区间（标准化后）颜色建议含义
极高80 - 100■ 深红优先更新区
高60 - 80■ 橙重点更新区
中40 - 60■ 黄一般更新区
低20 - 40■ 绿暂缓更新区
偏低0 - 20■ 蓝暂不更新区

5.4 评价结果关联宗地

原理：将潜力栅格值统计到宗地面要素上，每个宗地获得代表性潜力分值。

ArcEngine 实现：

  • GP 工具 Zonal Statistics as Table：以宗地为区域，潜力栅格为值
  • 统计方式：MEAN（均值）或 MAX（最大值），推荐 MEAN
  • JoinField 将统计结果写回宗地属性表
  • 新增字段：POTENTIAL_SCORE（总分）、MOTIV_SCORE（动力分）、FEASIB_SCORE（可行分）、POTENTIAL_LEVEL（等级）
  • 按 POTENTIAL_SCORE 分级渲染宗地图层（成果图2）

5.5 验证校核

原理：将系统评价结果与已知已更新宗地进行对比，检验模型合理性。

  • 导入「已更新宗地」图层（带更新状态字段）
  • 对比已更新宗地的潜力得分分布：已更新地块应集中在高/极高等级
  • 若偏差较大，通过 M8 权重配置模块调整准则层权重，重新计算
  • 记录校核意见，生成验证报告
